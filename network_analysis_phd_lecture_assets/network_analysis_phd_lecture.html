Why Networks? Motivation for Quant Methods

Networks are everywhere in social science: friends in a cohort, journalists citing sources, influencers mentioning brands, legislators cosponsoring bills, words co-occurring within tweets, or research teams coauthoring across institutions. Each example involves entities and ties that carry information beyond independent rows of a data frame.

When we assume rows are independent, we risk averaging away contagion, peer effects, or structural constraints. Modeling the ties directly preserves who can reach whom, which brokers sit across factions, and which ideas travel quickly or slowly. That perspective lets us connect back to regression thinking: network measures become predictors or outcomes that summarize how embedded each node is.

Dr. Vargo's own work illustrates these stakes clearly. His MSDS Marketing Text Analytics course builds multi-layer mention networks from @ATT, @TMobile, and @verizon Twitter archives and semantic networks that track which issues co-occur in brand conversations. His political communication collaborations (e.g., Network Agenda Setting) create issue-pair networks linking how campaigns, news outlets, and citizens couple topics like economy–jobs or healthcare–costs. We will draw on these same scenarios throughout the lecture so the concepts remain grounded in real research assets.

Treating ties as independent observations: Imagine stacking every mention on social media as a row, ignoring who mentioned whom. You could model counts, but you lose the possibility that a single influential user mentions multiple brands and redirects attention.
Respecting network structure: Build a mention network, summarize each user's degree, betweenness, or community membership, and relate those structural positions to campaign outcomes. Dependencies remain explicit, so you can discuss clustering, contagion, and sampling boundaries.

Meta scan of the advertising literature: Dr. Vargo's Social Media Network Analysis project (network_analysis_phd_lecture_assets/assets/social_media_network_analysis/JIAD_SocialNetworkCites.docx) used citation, co-citation, and network analysis on more than 100,000 co-citations covering 289 journals. The bibliometric clusters—word-of-mouth/diffusion, uses and gratifications, networks/networking, marketing/advertising/promotion, and methodology/statistics—show how deeply network reasoning already anchors strategic communication scholarship. The analysis also highlighted Journal of Advertising Research, Journal of Interactive Advertising, Journal of Interactive Marketing, and Public Relations Review as recurring hubs, reinforcing why network measures and visualization literacy matter for PhD students heading into advertising-focused research careers.

As an instructor, pause here and invite students to name a network they have implicitly analyzed in prior projects (e.g., citing authors, board memberships, or semantic co-occurrences). This primes them to see networks not as exotic math but as structured data frames.

Network Building Blocks: Nodes, Edges, and Types of Networks

  • Nodes (vertices): Actors, organizations, words, posts, or any discrete entity. Ego networks center on one node and its immediate ties; whole networks include all nodes in the defined boundary.
  • Edges (links): The relationships (friendship, mention, co-attendance, co-word). They can be directed (A mentions B) or undirected (A and B co-occur), weighted (counts, strengths), or signed (positive vs negative sentiment).
  • Adjacency: Nodes are adjacent if they share an edge. Adjacency lists enumerate neighbors of each node; adjacency matrices store ties as 0/1 (or weights) in a square array.
  • One-mode vs two-mode: One-mode networks connect entities of the same type (journalist-to-journalist). Two-mode (bipartite) networks connect different types (journalists-to-topics). Projecting a two-mode network onto one mode (e.g., topics connected if they appear in the same press release) is common in semantic and agenda-setting research.

Toy example: Consider five graduate students attending three workshops. A two-mode people–event network links each student to attended workshops. Projecting into a one-mode student network reveals who likely interacts outside class.

The R snippet below builds a directed mention-style toy network, prints both adjacency matrix and edge list, and plots the result. Toggle it when you are ready to walk through the syntax.

Representing Networks Mathematically

Adjacency matrix: Square matrix where entry aij equals 1 (or a weight) if there is a tie from node i to node j. Symmetric for undirected networks.

Adjacency list: For each node, record its neighbors (useful for sparse graphs). For example, A: {B, C}, B: {C}.

Edge list / long format: Data frame with columns such as from, to, weight, and optional time or attributes. This is how most CSV exports store networks for Gephi, igraph, or NetworkX.

Describing Networks: Global Structure

Descriptive measures help us compare networks before fitting sophisticated models:

  • Degree (k): Number of ties incident to a node (separate in/out for directed graphs). Degree distribution shows how popularity spreads. Formula: ki = ∑j aij.
  • Paths and geodesic distance: A path is a sequence of edges connecting nodes; the geodesic distance between nodes is the length of the shortest path. Network diameter is the longest geodesic.
  • Components: Maximal sets of nodes mutually reachable. A disconnected node sits in its own component.
  • Density: Proportion of realized edges out of all possible edges (for undirected networks: 2m / (n(n−1))).
  • Clustering coefficient: Probability that two neighbors of a node are connected (local) or average triadic closure (global). Indicates transitivity.

The Zachary Karate Club network (34 members) is a canonical example for measuring these properties.

library(igraph)

g <- make_graph("Zachary")

summary(karate_degree)

hist(karate_degree,
     breaks = 10,
     col = "#6baed6",
     main = "Degree distribution: Karate Club",
     xlab = "Degree")

cat("Density:", round(g_density, 3),"\n")

Centrality: Who or What is Important in a Network?

Centrality measures capture different ideas of importance:

  • Degree centrality: Popularity or activity; nodes with many ties can broadcast widely.
  • Betweenness centrality: Brokerage; nodes on many shortest paths can broker information. Think of communication strategists connecting departments.
  • Closeness centrality: How near a node is, on average, to others. Useful for diffusion speed intuition.
  • Eigenvector centrality: Influence that rewards being connected to other influential nodes (prestige).
  • PageRank: Like eigenvector, but damped random walk originally for web pages; handles directed networks and sinks gracefully.

This mirrors Dr. Vargo's carrier mention networks, where degree pinpoints super-fans (e.g., @denvermom talking to every carrier), betweenness identifies policy hubs (e.g., @techpolicyhub bridging infrastructure and customer care), and PageRank highlights which verified handles hold sustained attention even if they post infrequently.

Network Patterns and Models: Random, Small-world, Scale-free

Recognizing structural patterns helps you choose appropriate null models and theoretical frames. Dr. Vargo's brand dashboards regularly compare observed graphs to baseline simulations to argue whether a carrier conversation is unusually clustered (small-world) or dominated by a handful of hubs (scale-free).

  • Erdős–Rényi random graphs: Each possible edge is present with probability p. Degrees follow a binomial/Poisson-like distribution. Good baseline for testing whether observed clustering or hubs are surprising.
  • Small-world networks: High clustering like a lattice but short average path length due to a few long ties. Useful for modeling information spread in local communities with occasional jumps.
  • Scale-free networks: Heavy-tailed degree distributions with hubs produced via preferential attachment (the "rich get richer"). Social media follower graphs and semantic networks of popular topics often look scale-free.

Edges appear scattered without obvious hubs. Degree distribution is narrow, so most nodes look similar.

Notice clustered neighborhoods plus a few shortcuts. Communication cascades can traverse quickly despite local clustering.

A few hubs dominate (think mega-influencers or viral issues). Degree distributions follow a power-law, as shown earlier.

The R code compares three simulated graphs. Encourage students to tweak n or parameters to see when patterns blur.

Communities and Clustering

Communities (modules) are groups of nodes more densely connected internally than externally. Detecting them reveals factions, issue blocs, or audiences.

Modularity (Q): Measures how well a partition separates dense groups. Optimizing modularity (e.g., Louvain algorithm) is a common approach.

The Karate Club split famously aligned with two instructors, illustrating how structural fractures can predict real-world outcomes—just as Dr. Vargo's mention dashboards expose clusters of privacy advocates versus product promoters within telecom conversations.

modularity(comm)

table(membership(comm))

Network Agenda Setting on Twitter (Guo & Vargo, 2015)

Dr. Vargo's collaboration with Lei Guo—The Power of Message Networks: A Big-Data Analysis of the Network Agenda Setting Model and Issue Ownership (network_analysis_phd_lecture_assets/assets/guo_vargo_2015/Guo_Vargo_2015_Power_of_Message_Networks.pdf)—is the field story he tells whenever students ask, “Do these methods scale beyond toy graphs?” The team streamed 70 million tweets from August 1 through Election Day 2012, isolated 22 GB of usable data, and hand-tagged 46,003 ideologically self-described users (19,509 liberals; 26,494 conservatives). Every tweet mentioning one of the 16 issue lexicons (tax, jobs, deficit, economy, foreign affairs, immigration, health care, public order, LGBT, abortion, environment, energy, education, role of government, middle class, welfare) was converted into an ego network with Obama or Romney at the center and issues at the periphery. Sentistrength scores then filtered those networks into “associative” (any tie) versus “competent” (positive tie) versions. Quadratic assignment procedure (QAP) regressions compared Period 3 citizen networks (Oct 23–Nov 6) to Period 2 media or campaign networks (Sep 1–Oct 22).

Networks and Regression: Using Network Measures in Quantitative Models

Network measures can play two roles:

  • Predictors: Include degree, centrality, or community membership as covariates to predict outcomes like adoption, sales, or sentiment.
  • Outcomes: Model whether centrality changes when running interventions (e.g., does training increase a support agent's betweenness?).

Dependency warning: Standard regression assumes independent errors. Nodes in the same network share neighbors, so standard errors can be biased. Strategies include permutation tests, network autocorrelation models, or multi-level structures. At minimum, discuss these dependencies transparently.

library(igraph)
set.seed(42)

g <- sample_gnp(n = 30, p = 0.12)
deg <- degree(g)

adopt_prob <- plogis(-1 + 0.15 * deg)
adopted <- rbinom(30, 1, adopt_prob)

data_df <- data.frame(node = 1:30, degree = deg, adopted = adopted)

model <- glm(adopted ~ degree, data = data_df, family = binomial())
summary(model)

predict_df <- data.frame(degree = 0:10)
predict_df$prob <- predict(model, newdata = predict_df, type = "response")
predict_df

Interpretation: A positive coefficient on degree suggests each additional tie increases the log-odds of adoption. Emphasize that this does not prove causality; maybe already-enthusiastic nodes create both more ties and more adoptions. Encourage robustness checks (e.g., shuffle edges or run quadratic assignment procedures).

Pitfalls and Practical Issues

  • Boundary specification: Decide who is in the network (time window, geographic scope, platform). Missing actors bias centrality.
  • Missing edges: APIs or surveys rarely capture all ties. Document sampling processes and sensitivity analyses.
  • Direction & time: Static snapshots hide evolution. Consider longitudinal slices or dynamic models when behavior shifts.
  • Causality vs association: Centrality correlates with outcomes but does not, by itself, establish causal influence.
  • Data hygiene: Deduplicate users, correct encoding (UTF-16 exports in the social listening datasets), and anonymize sensitive handles when needed.
DoDon't
Document thresholds (e.g., users with >2 mentions) so analyses are reproducible.Filter opportunistically without noting how it changes density.
Check whether directed edges should be symmetrized before computing undirected measures.Mix directed and undirected interpretations in the same plot.
Use null models (random, degree-preserving) to contextualize clustering or hubs.Assume every observed pattern is meaningful without comparison.
Store raw edge lists plus derived metrics for reproducibility.Keep only the plotted layout coordinates.

Check Your Understanding

Use these quick questions as formative checks during class or as a post-lecture quiz. Click each button to reveal the explanation.

Q1: You have a directed mention network where most users mention only one brand, but one activist mentions five. Which centrality best captures their role as an information spreader?

  • A. Betweenness
  • B. Degree (out-degree)
  • C. Closeness
  • D. PageRank
Answer: B. Out-degree counts how many distinct brands they mention, indicating broadcasting potential. Betweenness would matter if they brokered between otherwise disconnected factions.

Q2: Which data structure is most compact for a sparse network with 10,000 nodes and only 15,000 edges?

  • A. Full adjacency matrix
  • B. Adjacency list or edge list
  • C. Raster image
  • D. Degree sequence only
Answer: B. Adjacency lists or edge lists store only existing edges, saving space and computation.

Q3: A bipartite network links news outlets to issues. To see how issues co-occur, what operation should you perform?

  • A. Randomly delete news outlets
  • B. Project the bipartite network onto the issue mode
  • C. Compute eigenvector centrality on the outlets
  • D. Convert to a time series
Answer: B. Project onto the issue mode so two issues connect when they appear in the same outlet, then analyze the resulting issue–issue network.

Q4: In a semantic network, you observe many triangles (word triples) about climate, energy, and jobs. Which descriptive measure captures this pattern?

  • A. Density
  • B. Clustering coefficient
  • C. Degree centrality
  • D. PageRank
Answer: B. The clustering coefficient quantifies the prevalence of closed triads or triangles.

Q5: A network has two dense subgroups with only a couple of ties between them. Which centrality should you highlight if you care about the few nodes who bridge those groups?

  • A. Betweenness
  • B. Degree
  • C. Closeness
  • D. Eigenvector
Answer: A. Betweenness centrality flags nodes that sit on many shortest paths between clusters, indicating brokerage.

Q6: Suppose you run a regression predicting customer advocacy from PageRank. Why must you mention network dependence in your model write-up?

  • A. Because PageRank is always zero
  • B. Because nodes connected in the network may have correlated errors, violating independence assumptions
  • C. Because regressions cannot include centrality scores
  • D. Because PageRank is undefined for directed graphs
Answer: B. Shared neighbors induce correlated outcomes, so you should discuss dependencies, resampling, or robust errors.

Q7: If an adjacency matrix is symmetric, what can you infer about the network?

  • A. It is directed
  • B. It is undirected (ties are mutual)
  • C. It has self-loops
  • D. It is weighted
Answer: B. Symmetry implies aij = aji, so the network is undirected.

Q8: Scenario: You find that density is very low, but closeness scores vary widely. What classroom discussion question could you pose?

  • A. Should we drop sparse nodes?
  • B. How can a sparse network still enable fast reach for some nodes via shortcuts?
  • C. Are densities and closeness identical?
  • D. Should we stop measuring centrality?
Answer: B. Even sparse graphs can contain shortcut paths that give some nodes high closeness, inviting discussion about small-world structure.
